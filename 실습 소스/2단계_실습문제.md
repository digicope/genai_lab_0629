# n8n 2단계 Workflow 실습 (중급 · 가입 불필요)

> **선수 과목:** `n8n_기초_실습` 완료 권장  
> **특징:** 기초 실습보다 **노드 수·로직이 복잡**하며, 여러 개념을 조합합니다  
> **사용 API:** [JSONPlaceholder](https://jsonplaceholder.typicode.com) (무료 · API Key 불필요)  
> **답안 위치:** `답안/` 폴더 개별 JSON → n8n **Import from File**

---

## 사전 준비

| 항목 | 내용 |
|------|------|
| n8n | Cloud 또는 로컬 설치 |
| Credential | **불필요** |
| 선행 학습 | Manual Trigger, HTTP Request, Set, IF, Merge, Code |

### 2단계에서 새로 배우는 것

- **연쇄 API 호출** — 이전 노드 결과로 다음 URL 구성
- **Filter** — 조건에 맞는 아이템만 통과
- **Switch** — 여러 갈래 분기
- **Webhook + Respond to Webhook** — 간단한 API 엔드포인트 만들기
- **노드 간 데이터 참조** — `$('노드이름')` Expression
- **Aggregate + Code** — 여러 데이터를 리포트로 합치기

---

## 실습 1. 연쇄 API 호출 — 게시글 → 작성자 조회

### 목표

게시글을 먼저 가져온 뒤, `userId`로 작성자 정보를 **연속 호출**합니다.

### 요구사항

```
Manual Trigger → HTTP Request(게시글) → HTTP Request(작성자) → Edit Fields
```

| 노드 | 설정 |
|------|------|
| **HTTP Request** | GET `https://jsonplaceholder.typicode.com/posts/1` |
| **HTTP Request1** | GET `https://jsonplaceholder.typicode.com/users/{{ $('HTTP Request').item.json.userId }}` |
| **Edit Fields** | 아래 3개 필드 생성 |

| 필드 | Expression |
|------|------------|
| `postTitle` | `={{ $('HTTP Request').item.json.title }}` |
| `authorName` | `={{ $json.name }}` |
| `authorEmail` | `={{ $json.email }}` |

### 확인 사항

- [ ] `postTitle`에 게시글 제목이 있는가?
- [ ] `authorName`, `authorEmail`에 작성자 정보가 있는가?
- [ ] HTTP Request1 URL에 **이전 노드 참조 Expression**을 사용했는가?

### 힌트

- `$('HTTP Request')` — 특정 노드의 출력 데이터 참조
- 두 번째 HTTP Request는 첫 번째 결과의 `userId`에 의존합니다

## 실습 2. Filter — 조건에 맞는 데이터만 추출

### 목표

전체 게시글 중 **userId가 1인 글만** 걸러낸 뒤 5건으로 제한합니다.

### 요구사항

```
Manual Trigger → HTTP Request → Filter → Limit → Edit Fields
```

| 노드 | 설정 |
|------|------|
| **HTTP Request** | GET `https://jsonplaceholder.typicode.com/posts` |
| **Filter** | `userId` **is equal to** `1` |
| **Limit** | Max Items: `5` |
| **Edit Fields** | `postId`, `title`, `userId` 필드만 유지 |

### 확인 사항

- [ ] Filter 이후 모든 아이템의 `userId`가 `1`인가?
- [ ] 최종 출력이 최대 5건인가?

### 힌트

- Filter 노드는 IF와 달리 **조건에 맞지 않는 아이템을 제거**합니다
- IF는 분기, Filter는 통과/차단입니다

---

## 실습 3. Switch — 완료/미완료 할 일 분류

### 목표

할 일 목록을 **완료(true) / 미완료(false)** 로 나누어 각각 다른 메시지를 만듭니다.

### 요구사항

```
Manual Trigger → HTTP Request → Switch → Edit Fields (완료)
                                      → Edit Fields1 (미완료)
```

| 노드 | 설정 |
|------|------|
| **HTTP Request** | GET `https://jsonplaceholder.typicode.com/todos` |
| **Switch** | Rules: `{{ $json.completed }}` **is true** / **is false** |
| **Edit Fields** (완료) | `status` = `완료`, `label` = `={{ $json.title }}` |
| **Edit Fields1** (미완료) | `status` = `미완료`, `label` = `={{ $json.title }}` |

### 확인 사항

- [ ] `completed: true` 항목이 **완료** 분기로 가는가?
- [ ] `completed: false` 항목이 **미완료** 분기로 가는가?
- [ ] 두 분기 모두 `status`, `label` 필드를 갖는가?

### 힌트

- Switch는 IF의 다중 분기 버전입니다
- todos API는 200건의 샘플 데이터를 반환합니다

## 실습 4. Webhook API — 인사 엔드포인트 만들기

### 목표

외부에서 호출 가능한 **간단한 API**를 만듭니다. (별도 서버 가입 불필요, n8n Webhook URL 사용)

### 요구사항

```
Webhook → Edit Fields → Respond to Webhook
```

| 노드 | 설정 |
|------|------|
| **Webhook** | Method: `POST`, Path: `greet`, Response Mode: **Using 'Respond to Webhook' Node** |
| **Edit Fields** | `greeting` = `={{ '안녕하세요, ' + $json.body.name + '님!' }}` |
| **Respond to Webhook** | Respond With: `JSON`, Body: `={{ { "message": $json.greeting } }}` |

### 테스트 방법

1. Webhook 노드에서 **Test URL** 복사
2. 워크플로우 **Activate** (또는 Listen for test event)
3. 아래 요청 전송:

```bash
curl -X POST {Webhook_URL} -H "Content-Type: application/json" -d "{\"name\": \"홍길동\"}"
```

### 확인 사항

- [ ] 응답 JSON에 `{ "message": "안녕하세요, 홍길동님!" }` 형태가 오는가?
- [ ] Webhook Response Mode가 Respond to Webhook 노드로 설정되었는가?

### 힌트

- POST body는 `$json.body`로 접근합니다
- 테스트 시 n8n Cloud는 **Activate** 후 Production URL 사용

## 실습 5. 댓글 리포트 — Aggregate로 목록 합치기

### 목표

게시글 댓글을 가져와 **하나의 텍스트 리포트**로 합칩니다.

### 요구사항

```
Manual Trigger → HTTP Request → Limit → Code → Aggregate → Edit Fields
```

| 노드 | 설정 |
|------|------|
| **HTTP Request** | GET `https://jsonplaceholder.typicode.com/comments?postId=1` |
| **Limit** | Max Items: `5` |
| **Code** | 각 댓글을 `line` 필드로 변환 (아래 로직) |
| **Aggregate** | `aggregateAllItemData` |
| **Edit Fields** | `report` = 댓글 목록 문자열 |

**Code 노드 로직**

```javascript
const { name, body } = $input.first().json;
return [{ json: { line: `- ${name}: ${body.substring(0, 40)}...` } }];
```

**Edit Fields `report` Expression**

```
={{ '📝 댓글 리포트 (postId=1)\n\n' + $json.data.map(item => item.line).join('\n') }}
```

### 확인 사항

- [ ] `report`에 5개 댓글 요약이 줄바꿈으로 나열되는가?
- [ ] Aggregate 이후 `$json.data` 배열을 사용했는가?

## 실습 6. 종합 — 게시글 상세 리포트

### 목표

게시글·작성자·댓글을 **순차 호출**한 뒤 Code 노드에서 하나의 리포트로 만듭니다. (2단계 종합 과제)

### 요구사항

```
Manual Trigger
  → HTTP Request (posts/1)
  → HTTP Request1 (users/{userId})
  → HTTP Request2 (comments?postId=1)
  → Limit (3건)
  → Code (종합 리포트)
```

| 노드 | URL |
|------|-----|
| **HTTP Request** | `https://jsonplaceholder.typicode.com/posts/1` |
| **HTTP Request1** | `https://jsonplaceholder.typicode.com/users/{{ $('HTTP Request').item.json.userId }}` |
| **HTTP Request2** | `https://jsonplaceholder.typicode.com/comments?postId={{ $('HTTP Request').item.json.id }}` |
| **Limit** | Max Items: `3` |
| **Code** | 아래 리포트 형식으로 `report` 필드 생성 |

**리포트 형식**

```
=== 게시글 리포트 ===
제목: {게시글 제목}
작성자: {이름} ({이메일})

--- 댓글 (상위 3건) ---
- {댓글작성자}: {댓글내용 30자}...
...
```

### 확인 사항

- [ ] 3개의 HTTP Request가 **순차 연결**되어 있는가?
- [ ] Code에서 `$('HTTP Request')`, `$('HTTP Request1')` 참조를 사용했는가?
- [ ] 댓글은 Limit 3건 기준으로 리포트에 포함되는가?

### 힌트

```javascript
const post = $('HTTP Request').first().json;
const user = $('HTTP Request1').first().json;
const comments = $input.all().map(i => i.json);

const commentLines = comments
  .map(c => `- ${c.name}: ${c.body.substring(0, 30)}...`)
  .join('\n');

return [{
  json: {
    report: `=== 게시글 리포트 ===\n제목: ${post.title}\n작성자: ${user.name} (${user.email})\n\n--- 댓글 (상위 3건) ---\n${commentLines}`
  }
}];
```

## Import 방법

1. n8n → **Workflows** → **⋯** → **Import from File**
2. `답안/` 폴더에서 해당 JSON 선택
3. **Test workflow** 실행 (실습 4는 Activate 후 Webhook 테스트)

| 실습 | 답안 파일 |
|------|-----------|
| 1 | `답안/실습1_연쇄_API.json` |
| 2 | `답안/실습2_Filter.json` |
| 3 | `답안/실습3_Switch.json` |
| 4 | `답안/실습4_Webhook.json` |
| 5 | `답안/실습5_댓글_리포트.json` |

## 기초 vs 2단계 비교

| 구분 | 기초 실습 | 2단계 실습 |
|------|-----------|------------|
| 노드 수 | 2~6개 | 4~7개 |
| API 호출 | 단일·병렬 | 연쇄·순차 |
| 분기 | IF (2갈래) | Switch (다중) |
| 데이터 참조 | `$json` | `$('노드명')` |
| 출력 | 필드 추출 | 리포트·API 응답 |
